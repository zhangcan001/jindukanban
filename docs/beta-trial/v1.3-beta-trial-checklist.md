# v1.3-beta 真实 Excel 试用清单

当前版本：v1.3-internal

阶段目标：用真实项目 Excel 验证 v1.3 是否具备进入 v1.3-beta 的可用性。

原则：

- 不新增 v1.4 功能。
- 每次只处理真实试用暴露的问题。
- 先复现、分类、判断优先级，再做小范围修复。
- 修复后必须运行 `python -m pytest` 和 `npm run build`。
- 不做大规模重构，不改变当前架构。

## 四、重点观察问题

### 字段识别

- WBS / WBS编码 / 工作分解结构编码是否识别为 `wbs_code`
- 任务编码 / 清单编码 / 编号是否识别为 `task_code`
- 工作内容 / 施工内容 / 任务名称 / 子项是否识别为 `task_name`
- 计划完成率 / 应完成率是否识别为 `planned_percent`
- 实际完成率 / 进度百分比是否识别为 `actual_percent`
- 本周完成量 / 本期完成是否识别为 `period_quantity`
- 累计完成量 / 已完成是否识别为 `cumulative_quantity`

### 数据解析

- `0.58`、`58%`、`58` 的百分比语义是否符合用户预期
- 日期格式是否能被正确识别
- 负数、超 100%、非法日期是否能产生清晰提示
- 地下楼层、专业简称是否能标准化

### 汇总行

- 合计、小计、总计、汇总是否跳过
- 本页合计、本周合计、专业小计、区域小计是否跳过
- 正常任务名如“合计箱安装”是否保留

### 统计口径

- overview 与 group-by 口径是否符合现场理解
- 单位混杂 warning 是否清晰
- 工程量百分比、平均百分比、加权百分比是否符合预期

### 报表

- 报表是否能直接用于会议
- 字段顺序是否可接受
- 计划基线名称是否清楚
- 明细中是否混入合计行或无效行

## 五、问题分级建议

| 级别 | 定义 | 处理策略 |
|---|---|---|
| P0 | 启动失败、无法导入、数据污染、报表无法打开 | 立即修复 |
| P1 | 主流程可走但统计明显错误、任务匹配严重错误 | v1.3-beta 前修复 |
| P2 | 不阻塞试用，影响效率或局部体验 | 视数量决定是否修 |
| v1.4 候选 | 新能力、新架构、较大交互升级 | 记录，不在 beta 试用期实现 |

## 六、每次修复后的回归要求

必须运行：

```powershell
cd backend
.\.venv\Scripts\python.exe -m pytest

cd ..\frontend
npm run build
```

按问题影响范围补充最小回归：

- 导入问题：重新上传同一真实 Excel，跑 parse / validate / confirm / publish。
- 看板问题：检查 overview、group-by、trend、delayed-ranking。
- 报表问题：重新导出并打开对应报表。
- 项目删除或配置问题：跑 health check 和最小项目流程。

## 七、v1.3-beta 进入标准

满足以下条件后建议标记 v1.3-beta：

- 至少 5 类真实 Excel 试用完成。
- P0 问题清零。
- v1.3-beta 必修 P1 问题清零。
- pytest 通过。
- npm run build 通过。
- 主流程导入、发布、看板、趋势、预警、报表导出通过。
- README 已更新试用结论和已知限制。
